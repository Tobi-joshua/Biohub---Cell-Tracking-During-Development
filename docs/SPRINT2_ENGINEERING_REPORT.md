## Phase 2 — V4 vs V5 comparison

Commits: **v1.4** = `8c19f55`, **v1.5** = `6682370`

| File | Change | Why (intent) | Expected impact | Actual LB |
|------|--------|--------------|-----------------|-----------|
| `config.py` | `gap_close_enabled=True`, `prune_soft_neighbors=True`, `div_symmetry_weight=0.35` | Recover missed links; keep near-neighbor orphans; penalize asymmetric divisions | ↑ recall | **↓ 0.659→0.648** |
| `tracking.py` | `close_frame_gaps()` — link `t-2→t` within 15 µm | Bridge one-frame dropouts | ↑ edge recall | Extra false edges hurt edge Jaccard |
| `tracking.py` | Soft orphan pruning within 9 µm of linked nodes | Avoid deleting true cells near tracks | ↑ node recall | Extra unmatched nodes hurt precision |
| `tracking.py` | Division symmetry penalty in cost matrix | Reject implausible mitosis geometry | ↑ division precision | May drop true divisions |
| `detection.py` | Adaptive threshold retry bug fix + retry loop | Cap per-frame over-count | ↓ false positives | Changes counts vs frozen v4 |
| `tuning.py` | Expanded hyperparameter grid | Offline search | Neutral if search disabled | Neutral |

### Regression root cause (ranked, evidence-supported)

1. **Gap closing** — adds edges outside the 11 µm Hungarian gate; synthetic A/B shows **+57% edges** (95→149) with unchanged detections/frame.
2. **Soft orphan pruning** — retains nodes v4 would remove; synthetic A/B shows **+50% nodes** (109→163).
3. **Adaptive threshold retry** — now active when `use_adaptive_frame_threshold=True`; v4 notebook behavior had retry effectively inert on many frames.
4. **Division symmetry** — secondary; same division count on synthetic but may reorder daughter assignment on real data.

**V6 action:** `competition_v4_preset()` disables all four. Defaults in `Config` also set v1.5 flags OFF.

## Phase 3 — Error analysis

### Detection

| Issue | Metric effect |
|-------|---------------|
| False positives | ↓ node precision; spurious edges |
| False negatives | ↓ node recall; broken tracks |
| Merged peaks (dense clusters) | ↓ node recall; wrong topology |
| Duplicate peaks (NMS too loose) | ↓ node precision |
| Threshold too high/low | Shifts precision/recall tradeoff |
| Border artifacts | Localized FP/FN at volume edges |

### Tracking

| Issue | Metric effect |
|-------|---------------|
| Broken trajectories (no link) | ↓ edge recall |
| False Hungarian assignments | ↓ edge Jaccard |
| Gap-close long edges (15 µm) | ↓ edge Jaccard, wrong division candidates |
| Fragmented tracks | ↓ edge recall, isolated nodes pruned |
| Identity switches | ↓ edge precision on dense regions |

### Division inference

| Issue | Metric effect |
|-------|---------------|
| Missed divisions | ↓ division recall |
| False divisions (spurious 1→2 edges) | ↓ division precision |
| Weak heuristics (parent/sister gates) | Both precision and recall |
